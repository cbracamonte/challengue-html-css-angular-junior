1. Creación de Componentes
a. Crea un nuevo componente llamado saludo. Este componente debe mostrar un mensaje de saludo en el navegador.
b. Crea otro componente llamado contador. Este componente debe mostrar un número y tener botones de "Incrementar" y "Decrementar" que permitan modificar ese número.


2. Directivas
a. Crea una directiva applyStyles. Este directiva debe aplicar estilos de css a cualquier a un elemento de HTML
b. Crea otra directiva llamado transformString. Esta directiva debe aplicar la transformación de una cadena a mayúscula de un elemento de HTML


3. Data Binding y Reactive Forms
a. Crea un formulario de entrada de texto en un componente. Utiliza el enlace de datos de dos vías ([(ngModel)]) para mostrar el valor del campo de entrada en tiempo real en un párrafo debajo del campo.
b. Crea un formulario que solo contenga usuario y correo y estos cumplan con lo siguiente:
- Ambos campos deben ser obligatorios
- El usuario debe contener la palabra peakr
- El correo debe cumplir el formato de email
- El botón de guardar, debe deshabilitarse/habilitarse si cumple las 2 reglas
Nota: Los nombres de los componentes son a libertad propia


4. Inputs & Ouputs
Crea un componente llamado TareaListComponent (Listado de Tareas) que mostrará una lista de tareas.
Define una propiedad task que será un arreglo de objetos de tarea, donde cada objeto tendrá una descripción y un estado (completo o no completo).
a. Utiliza el decorador @Input para permitir que el componente padre pase el arreglo de tareas al componente TareaListComponent.
b. Renderiza las tareas en una lista y muestra si cada tarea está completa o no. Proporciona una forma de marcar una tarea como completa o incompleta. (Puedes aplicar buenas prácticas como trackBy)
c. Crea otro componente llamado TareaFormComponent que contendrá un formulario para agregar nuevas tareas. Define una propiedad nuevaTarea y un método agregarTarea() para agregar tareas a la lista.
d. Utiliza el decorador @Output para emitir eventos cuando se agrega una nueva tarea. El evento debe llevar consigo la nueva tarea que se agregó.

Objetivo:
- Cómo se comunican los componentes: TareaFormComponent debe emitir un evento cuando se agrega una nueva tarea.
- El evento debe llevar la nueva tarea como datos.
- TareaListComponent debe recibir este evento y agregar la nueva tarea al arreglo de tareas.
